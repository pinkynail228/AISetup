# Setups Hub (AISetup)

Реестр версионируемых **сетапов** для работы с ИИ-агентами. Сетап — это готовый набор инструкций под конкретный тип проекта: процесс с жёсткими ревью-гейтами, скилы, чек-листы, манифесты и шаблоны. Скачиваешь архив в пустой репозиторий — и агент (Codex или любой, понимающий AGENTS.md) ведёт проект по выстроенному процессу, останавливаясь на ревью.

**Витрина:** https://pinkynail228.github.io/AISetup/
**Машиночитаемый реестр:** https://pinkynail228.github.io/AISetup/registry.json

## Как скачать сетап

1. Через витрину — кнопка «Скачать .zip» (отдаёт архив из GitHub Releases).
2. Напрямую из [Releases](https://github.com/pinkynail228/AISetup/releases) — релизы вида `website-setup@1.0.0`.
3. Для агентов — `registry.json` содержит актуальные версии и прямые ссылки на архивы.

Дальше: распаковать в корень нового репозитория → заполнить `site/BRIEF.md` → сказать агенту `Read AGENTS.md and STATE.md, then start Phase 1.` Подробности — в `README.ru.md` каждого сетапа.

## Структура репозитория

```
setups/<id>/          ← сетапы; манифест каждого — SETUP.yaml
registry/             ← registry.json, агрегат манифестов (генерируется, руками не правится)
apps/web/             ← витрина (Next.js, статический экспорт → GitHub Pages)
.github/workflows/    ← CI: деплой витрины + авторелиз сетапов по бампу версии
AGENTS.md             ← мета-процесс обновления сетапов (для ИИ-агентов-мейнтейнеров)
```

## Как обновляются сетапы

Изменение сетапа = PR с бампом версии в `SETUP.yaml` и записью в changelog → ревью владельцем → merge в `main` → CI сам публикует релиз `id@version` с архивом и пересобирает витрину. Semver: **major** — изменился процесс, **minor** — новые скилы/чек-листы, **patch** — формулировки. Поле `owned_paths` в манифесте определяет, какие папки заменяются при обновлении сетапа в действующем проекте, а какие принадлежат проекту.
