# Definition of Done — Phase 1: Discovery & Strategy

Every box checked before closing the phase and moving to Phase 2.

- [ ] `site/strategy.md` exists and was written from `process/templates/strategy.template.md` — no section is blank or skipped.
- [ ] Every goal in the Goals section has at least one measurable success metric (a number, a rate, a deadline — not a feeling).
- [ ] At least one audience persona is defined; each persona has situation, motivation, and at least one objection recorded.
- [ ] The value proposition is written as one sentence that names the target audience, the outcome they get, and why this offering delivers it.
- [ ] Messaging pillars are 3–5 in number; each is one declarative sentence, not a label.
- [ ] Tone of voice lists at least 3 adjectives AND at least 3 explicitly banned words or phrases.
- [ ] Scope section names the target page set (a number or a range, not "TBD") and lists every page type anticipated.
- [ ] Site language is explicitly stated (e.g., "Russian", "English") — not left blank.
- [ ] Performance budgets are explicit numbers: Core Web Vitals targets (LCP, CLS, INP), Lighthouse minimums for Performance and Accessibility on mobile, and WCAG level (must be 2.2 AA or stricter).
- [ ] Every place the brief was silent, `site/strategy.md` lists a numbered assumption that can be individually vetoed.
- [ ] There is no lorem ipsum text anywhere in `site/strategy.md`.
- [ ] `STATE.md` is updated: Phase 1 status set to `done`, completion date recorded.
