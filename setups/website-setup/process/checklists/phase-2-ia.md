# Definition of Done — Phase 2: Information Architecture

Every box checked before requesting GATE 1.

- [ ] `site/sitemap.md` exists and was written from `process/templates/sitemap.template.md` — every page has a URL slug, a one-line purpose statement, and a navigation placement (header, footer, or orphan).
- [ ] The page set in `site/sitemap.md` matches the scope stated in `site/strategy.md` (count is within the agreed range; no undeclared pages added).
- [ ] Every page in the sitemap has a corresponding brief file at `site/page-briefs/<slug>.md` written from `process/templates/page-brief.template.md`.
- [ ] Every page brief contains: slug, purpose, primary audience (matching a persona from strategy), primary message as exactly one sentence, an ordered section outline with intent per section, and a primary CTA.
- [ ] Every page brief contains SEO intent: a target query, a draft title of 60 characters or fewer, and a draft meta description of 155 characters or fewer.
- [ ] No page brief has the same primary message as another page brief (each page has a distinct job).
- [ ] The internal linking plan table in `site/sitemap.md` lists every planned link between pages (source slug → target slug → anchor text intent) — the table is not empty.
- [ ] There is no lorem ipsum text anywhere in any IA artifact.
- [ ] Every assumption made without explicit input from the brief is listed and numbered in the relevant artifact so the human can veto it.
- [ ] `process/templates/review-request.template.md` has been filled and posted in chat for GATE 1.
- [ ] `STATE.md` status is set to `blocked-pending-review` with the artifact list for GATE 1 recorded.
- [ ] Marker `REVIEW_GATE_1` has been output in chat.
