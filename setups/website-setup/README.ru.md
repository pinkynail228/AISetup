# Website Build Setup v1.0.0 — сетап для создания сайтов с ИИ-агентом

Готовый набор инструкций, процессов и скилов, который превращает ИИ-агента (в первую очередь **OpenAI Codex**) в дисциплинированную веб-студию. Агент ведёт проект через **9 этапов с 4 обязательными ревью-гейтами**: сначала стратегия и тексты, потом — и только потом — визуал и вёрстка. На каждом гейте агент останавливается и ждёт вашего аппрува.

**Стек:** Next.js (App Router) + TypeScript + Tailwind CSS v4.

## Быстрый старт

1. Распакуйте архив в корень нового (пустого) репозитория и сделайте `git init`, если репозитория ещё нет.
2. Заполните `site/BRIEF.md` — можно на русском, можно частично. Чем конкретнее бриф, тем меньше итераций.
3. Запустите Codex в этом репозитории (CLI, IDE или веб) и отправьте первую команду:

   > Read AGENTS.md and STATE.md, then start Phase 1.

4. Дальше агент работает по пайплайну сам. Ваше участие — четыре ревью.

## Этапы и гейты

| Этап | Что происходит | Ваше участие |
|---|---|---|
| 1. Discovery & Strategy | Бриф → стратегия: цели, аудитория, тон, бюджеты качества | — |
| 2. Information Architecture | Карта сайта + бриф каждой страницы | **ГЕЙТ 1** — ревью стратегии и структуры |
| 3. Copywriting | Полные реальные тексты всех страниц | — |
| 4. Validation & Wireframes | Валидация текстов + LoFi-скелеты страниц | **ГЕЙТ 2** — ревью текстов (итерации — норма) |
| 5. Design Tokens | Палитра, типографика, сетки — как токены | — |
| 6. UI Primitives | Кнопки, карточки, поля… + страница /styleguide | **ГЕЙТ 3** — ревью визуального языка |
| 7. Page Assembly | Сборка страниц из утверждённых текстов и примитивов | — |
| 8. QA & Accessibility | Lighthouse, доступность, адаптивность, отчёт | **ГЕЙТ 4** — финальное ревью сайта |
| 9. Launch | Чек-лист запуска, деплой, smoke-проверки | приёмка |

## Как отвечать агенту на гейте

Агент выведет review request (что сделано, ключевые решения, допущения, вопросы) и маркер `REVIEW_GATE_N`, после чего остановится.

- **«approved» / «утверждаю»** — гейт закрыт, агент идёт дальше (аппрув фиксируется в `STATE.md`);
- **список правок** — агент внесёт их в рамках текущих этапов и снова запросит ревью;
- вопросы можно задавать свободно, на русском — артефакты процесса агент ведёт на английском, тексты сайта — на языке из стратегии.

Если агент порывается проскочить гейт — напомните: *«Follow the review gate protocol in AGENTS.md»*.

## Состав архива

```
AGENTS.md            ← ядро: правила, карта этапов, протокол гейтов (читается агентом первым)
MANIFESTO.md         ← 10 принципов качества; решает конфликты
STATE.md             ← живой статус проекта: этап, аппрувы (ведёт агент)
SETUP.yaml           ← версия и метаданные сетапа
README.ru.md         ← этот файл
site/BRIEF.md        ← ваш бриф (единственный файл, который заполняете вы)
process/
  PIPELINE.md        ← подробное описание всех 9 этапов
  checklists/        ← Definition of Done на каждый этап (9 файлов)
  templates/         ← шаблоны артефактов: стратегия, sitemap, бриф страницы,
                       тексты, вайрфрейм, review request
.agents/skills/      ← 9 скилов «как делать этап» (нативный формат скилов Codex)
docs/
  stack-conventions.md ← конвенции кода: Next.js, Tailwind v4, токены, доступность
```

Артефакты вашего проекта появятся по ходу работы: `site/strategy.md`, `site/copy/`, `site/wireframes/`, `design/tokens.json`, `app/`, `components/`, `reports/`.

## Ключевые принципы (из манифеста)

- **Content before chrome** — ни одного визуального решения до утверждения текстов. Никакого lorem ipsum.
- **Токены — инфраструктура** — каждый цвет и отступ берётся из токена; ребрендинг = правка токенов, а не рефакторинг.
- **Гейты — жёсткие стопы** — агент никогда не утверждает сам себя; молчание ≠ аппрув.
- **Состояние в файлах** — проект можно продолжить с любой новой сессии: всё в `STATE.md` и артефактах.

## Совместимость и обновление

- **Codex** — основной таргет (формат AGENTS.md + нативные скилы `.agents/skills/`).
- Подойдёт любому агенту, понимающему AGENTS.md. Для **Claude Code** создайте `CLAUDE.md` с одной строкой: `Read AGENTS.md and follow it.`
- Версия сетапа — в `SETUP.yaml`. Обновление: замените `process/`, `.agents/`, `docs/`, `AGENTS.md`, `MANIFESTO.md` файлами новой версии — артефакты вашего проекта (`site/`, `design/`, `app/`) не затрагиваются.

---

*Website Build Setup — первый сетап серии. Формат рассчитан на расширение: лендинги, блоги, SaaS-приложения и другие типы проектов могут использовать ту же структуру (AGENTS.md + манифест + пайплайн с гейтами + скилы + чек-листы).*
